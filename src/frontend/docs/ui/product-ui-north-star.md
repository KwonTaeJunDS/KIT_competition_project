# 한국사 오답노트 RAG OS

## Product UI North Star v1

### 1. 왜 이 프로젝트를 하는가

이 프로젝트는 단순한 문제 풀이 앱이 아니다.

이 프로젝트의 목적은:

- 선생님이 `한국사 지식 구조`를 직접 만들거나 자동으로 구축할 수 있게 하고
- 학생이 그 구조 안에서 `무엇을 모르고`, `어디서 연결이 끊겼는지`, `왜 이해가 안 되었는지`를 추적 가능하게 만들고
- 그 결과가 다시 선생님에게 돌아가 `어떤 설명이 전달되지 않았는지`를 확인할 수 있게 만드는 것이다.

즉, 이 서비스는

**문제를 채점하는 서비스가 아니라, 전달되지 않은 개념과 흐름을 드러내는 한국사 학습 관리 OS**다.

---

### 2. 제품의 본질

이 제품은 두 사용자 층을 가진다.

#### A. 선생님

선생님은 지식을 “가르치는 사람”이 아니라,
`개념 관계망을 설계하고`, `학생의 이해 누수 지점을 추적하는 사람`이다.

선생님 화면의 본질은 다음이다.

- PDF, 정답지, 사료집, 프린트 등에서 개념을 추출
- 개념 간 관계를 자동 또는 수동으로 온톨로지화
- 학생별로 어떤 노드와 연결이 비었는지 확인
- 어떤 설명이 전달되지 않았는지 파악
- 다시 어떤 순서로 가르쳐야 할지 판단

#### B. 학생

학생은 단순히 맞고 틀리는 사람이 아니라,
`한국사 전체 흐름 안에서 자신의 이해가 어디서 끊겼는지`를 확인하는 사람이다.

학생 화면의 본질은 다음이다.

- 오늘 무엇을 먼저 봐야 하는지 판단
- 어떤 시대 흐름이 약한지 확인
- 어떤 연결이 끊겼는지 이해
- 문제, 복습, 오답을 따로 보지 않고 `구조적으로` 다시 학습
- 자신의 부족한 개념이 선생님에게도 전달된다는 신뢰를 가짐

---

### 3. 우리가 만들고 있는 경험

우리는 “한국사 앱”을 만드는 것이 아니라,

**한국사 개념 전달과 이해 누수를 추적하는 교육 운영 플랫폼**을 만들고 있다.

따라서 UI는 다음 질문에 답해야 한다.

- 지금 학생은 어디에서 막혔는가
- 이 막힘은 어떤 개념 연결의 문제인가
- 이 문제는 어느 시대 흐름의 문제인가
- 선생님은 이걸 어떻게 다시 설명해야 하는가

즉 UI는 “예쁨”보다 먼저
`판단`, `구조`, `전달`, `복원`을 지원해야 한다.

---

### 4. UI 역할 정의

#### 선생님 UI

선생님 UI는 `콘텐츠 관리 툴`이 아니라 `지식 구조 운영실`처럼 보여야 한다.

핵심 인상:

- 정교함
- 통제감
- 관계망
- 추적 가능성
- 전달 분석

핵심 기능 언어:

- 소스 업로드
- 자동 추출
- 개념 노드
- 관계 편집
- 학생별 이해 누수
- 전달 실패 지점
- 재설명 우선순위

#### 학생 UI

학생 UI는 `할 일 앱`도 아니고 `박물관풍 앱`도 아니다.

핵심 인상:

- 지금 무엇을 먼저 해야 하는지 분명함
- 내가 한국사 전체 속 어디쯤 있는지 보임
- 틀린 문제가 아니라 `끊긴 연결`이 보임
- 문제 풀이가 아니라 흐름 복원처럼 느껴짐

핵심 기능 언어:

- 오늘의 판단
- 중심 시대
- 끊긴 연결
- 복습 흐름
- 시대 지도
- 오답 기록

### 5. 디자인 노스스타

#### 공통

전체 제품은 다음 한 줄로 잠근다.

**Modern Education OS + Korean Historical Structure**

뜻:

- 전체 구조는 현대적인 운영 도구처럼 명확해야 한다
- 한국성은 장식이 아니라 구조와 질서에서 나와야 한다
- 한국사 느낌은 텍스트보다 관계, 선, 축, 기록성에서 느껴져야 한다

#### 선생님 쪽 디자인 방향

- 운영 도구
- 분석 대시보드
- 온톨로지 편집기
- 개념 그래프
- 전달 추적 보드

즉:

**정확하고, 밀도 높고, 통제 가능한 UI**

#### 학생 쪽 디자인 방향

- 판단 데스크
- 흐름 지도
- 기록형 오답노트
- 회독 관리

즉:

**이해를 복원하는 UI**

### 6. UI가 절대 놓치면 안 되는 것

#### 1. 선생님과 학생이 같은 구조를 보고 있어야 한다

학생이 보는 “끊긴 연결”은
선생님에게는 “전달 실패 지점”으로 보이게 연결되어야 한다.

#### 2. 시대와 개념은 따로 놀면 안 된다

한국사는 시대명만 보여준다고 학습이 되지 않는다.
`시대 -> 흐름 -> 사건 -> 개념 -> 오답`이 하나로 이어져야 한다.

#### 3. 오답은 기록이 아니라 진단이어야 한다

오답 수를 보여주는 것은 약하다.
어떤 개념 연결이 왜 끊겼는지를 보여줘야 한다.

#### 4. UI는 “관리”보다 “복원”의 느낌을 줘야 한다

특히 학생 화면은 생산성 앱처럼 보이면 안 된다.
“오늘 무엇을 정리하면 전체 흐름이 복원되는가”가 먼저 와야 한다.

### 7. 현재 기준으로 잠그는 방향

#### 학생 화면

- `/today`: 오늘의 판단과 중심 시대를 여는 데스크
- `/solve`: 가장 집중도가 높은 풀이 화면
- `/notes`: 끊긴 연결을 복원하는 기록 보관소
- `/review`: 잊힌 흐름을 되살리는 회독 큐

#### 선생님 화면

아직 본격 설계 전이지만 최종적으로는 아래가 필요하다.

- 소스 업로드
- 자동 개념 추출
- 관계망 편집
- 개념 그래프 보기
- 학생별 누수 분석
- 전달 실패 리포트

### 8. 아직 안 잠긴 질문

다음 질문은 답을 받아야 전체 UI 방향이 고정된다.

#### Q1. 선생님 화면의 중심은 무엇인가

가장 중요한 것이 무엇인지 잠가야 한다.

- 온톨로지 편집
- 소스 자동 추출
- 학생 이해 누수 분석
- 전달 실패 리포트

#### Q2. 학생 화면의 최종 감정 톤은 어디인가

- 더 시험형
- 더 학습형
- 더 기록형

현재는 `학습형 + 기록형` 쪽이 유력하다.

#### Q3. 학생 결과는 선생님에게 어떻게 전달되는가

- 자동 리포트형
- 학생별 대시보드형
- 개념 노드 heatmap형

이게 정해져야 학생 UI와 선생님 UI가 하나의 시스템으로 설계된다.

### 8-1. 우선순위 잠금

아래 항목은 현재 기준으로 우선순위를 잠근다.

#### 선생님 화면 1순위

**직접 관계망 편집**

뜻:

- 자동 온톨로지화도 중요하지만, 최종적으로 선생님이 직접 구조를 만지고 고칠 수 있어야 한다
- 선생님 UI의 중심은 업로드 화면이 아니라 `관계망을 다루는 작업면`이어야 한다
- 자동 추출은 시작점이고, 편집은 핵심 가치다

#### 학생 화면 감정 톤

**시험 대비형**

뜻:

- 학생 화면은 감성 앱보다 `실전 대비 도구`처럼 느껴져야 한다
- 다만 생산성 앱처럼 차갑게만 가지 않고, `흐름 복원` 개념을 같이 안고 간다
- 즉 기본 톤은 시험 대비형, 보조 성격은 학습 복원형이다

#### 선생님이 보는 핵심 단위

**학생별**

뜻:

- 리포트와 추적은 먼저 학생 단위로 읽혀야 한다
- 그 안에서 시대, 개념, 문항, 소스가 보조적으로 펼쳐지는 구조가 적합하다
- 즉 개념 그래프가 메인 홈이 아니라, 학생별 이해 상태가 메인 진입점이어야 한다

#### 제품의 가장 강한 한 줄 가치

**학생이 끊긴 흐름을 찾는다**

뜻:

- 이 제품은 맞고 틀림만 보여주는 것이 아니라, `어디서 흐름이 끊겼는지`를 보여줘야 한다
- 선생님에게도 결국 전달되지 않은 흐름이 보이도록 연결되어야 한다

#### 함께 유지해야 하는 보조 가치

위 우선순위를 잠그되, 아래 가치도 절대 버리지 않는다.

- 자료가 자동으로 개념망이 된다
- 선생님이 전달 안 된 개념을 찾는다
- 학생이 시대와 개념의 연결을 구조적으로 이해한다

즉, 제품의 주력 메시지는 하나로 가되 실제 기능 가치는 다층적으로 유지한다.

---

### 8-2. 진입 우선순위 잠금

#### 선생님 첫 화면

**학생별 이해 누수 현황**

뜻:

- 선생님이 가장 먼저 봐야 하는 것은 “지금 누구에게 무엇이 전달되지 않았는가”다
- 관계망 편집은 핵심 작업이지만, 첫 진입은 `학생별 상태 파악`이 우선이다
- 따라서 어드민 홈은 업로드 화면이 아니라 `학생별 이해 누수 대시보드`가 되어야 한다

#### 학생 첫 화면

**오늘 복습 우선**

뜻:

- 학생 홈은 먼저 “오늘 무엇부터 복원해야 하는가”를 보여줘야 한다
- 중심 시대와 끊긴 연결은 그 판단을 돕는 구조로 배치한다
- 즉 학생 홈의 첫 행동은 `복습 시작`이어야 한다

#### 어드민 시각 톤

**교육기관용 운영 대시보드**

뜻:

- 차갑고 추상적인 분석 툴보다 더 친숙해야 한다
- 그렇다고 일반 SaaS 관리페이지처럼 가벼워서도 안 된다
- 학교/학원/교사 환경에서 바로 쓸 수 있는 `운영 대시보드 + 구조 편집기` 톤이 맞다

---

### 9. 최종 잠금 문장

이 프로젝트는

**한국사 자료를 개념 구조로 바꾸고, 학생의 이해 누수를 추적해, 선생님이 다시 가르칠 지점을 발견하게 만드는 교육 운영 플랫폼**이다.

그래서 UI는 예쁜 것보다 먼저

- 구조가 보여야 하고
- 전달 실패가 보여야 하고
- 학습 흐름이 복원되어야 한다.
