# 한국사 학습 OS 테마 DNA 시스템 v1

## 목적

이 문서는 `/today`, `/notes`, `/review`, `/solve`에 적용한 미감과 구조를 페이지 개별 디자인이 아니라 **공통 테마 시스템**으로 고정한 기준서다.

목표는 다음과 같다.

1. 새로운 탭이나 컴포넌트가 생겨도 같은 세계관으로 즉시 확장 가능해야 한다.
2. 단순한 색상 모음이 아니라, 한국사 학습 제품만의 정체성 규칙을 제공해야 한다.
3. 누가 만들더라도 화면이 제각각 흩어지지 않고 같은 언어를 쓰게 해야 한다.
4. 현재 구현된 프론트의 공용 프리미티브와 실제 탭 역할을 기준으로 디자인 결정을 잠가야 한다.

---

# 1. 테마 한 줄 정의

> 현대적인 학습 인터페이스 위에, 한국의 기록문화·시간감·정갈한 위계 질서를 얹은 `Time Descent` 기반 학습 OS 테마

이 테마는 전통을 직접 묘사하지 않는다.
대신 아래를 UI 구조로 번역한다.

- 기록물의 정리감
- 시대 흐름의 깊이
- 여백과 선의 절제
- 기억 복원의 차분한 리듬
- 현재에서 과거로 내려가는 층위감

---

# 2. 핵심 정체성 DNA

## 2.1 이 테마가 주는 감정

사용자가 화면을 볼 때 느껴야 하는 감각은 다음 5가지다.

1. 정갈함: 정신없지 않고 정리된 느낌
2. 기록성: 정보가 가볍게 뜨지 않고 보관된 것 같은 느낌
3. 시간성: 시대 흐름이 살아 있는 느낌
4. 회복성: 틀린 것을 혼내는 화면이 아니라, 다시 이어주는 화면
5. 격식: 너무 장난스럽지 않고, 공부할 만한 무게감이 있는 느낌

## 2.2 이 테마가 피해야 하는 감정

1. 관광지 기념품 느낌
2. 박물관 이벤트 포스터 느낌
3. 과한 전통문양 장식 느낌
4. 평범한 SaaS 템플릿 느낌
5. 너무 무거워서 손이 안 가는 교육 툴 느낌

# 3. 디자인 철학

## 3.1 구조 우선

전통 문양보다 먼저 정보 구조가 한국적 질서감을 가져야 한다.

- 제목은 중심축이 분명해야 한다.
- 정보는 층위가 있어야 한다.
- 섹션은 공간처럼 느껴져야 한다.
- 위에서 아래로 갈수록 시간의 깊이가 더해져야 한다.

## 3.2 장식보다 질감

한국사 느낌은 장식이 아니라 재질감과 밀도로 만든다.

- 얇은 선
- 절제된 강조
- 낮은 채도
- 문서형 패널
- 보관함 같은 블록 구조

## 3.3 학습 흐름이 곧 미감

이 제품의 미학은 화면 장식이 아니라 학습 루프 자체에서 나온다.

- 오늘: 시간축 진입
- 오답: 기록 보관
- 복습: 기억 회복
- 학습: 다시 적용

즉, 학습 경로 자체가 UI 미학이 된다.

# 4. 제품 세계관과 탭 역할

## 4.1 제품 세계관

이 제품은 문제 풀이 앱이 아니라, **한국사 흐름을 기록하고 복원하는 학습 OS**다.

## 4.2 화면 해석 방식

각 화면은 단순 탭이 아니라 역할을 가진 공간으로 해석한다.

- `/today` = 시대 입구
- `/notes` = 기억 보관실
- `/review` = 회복실
- `/solve` = 판단실

향후 확장 예시는 아래처럼 잡는다.

- `/timeline` = 전체 사관 지도
- `/archive` = 누적 기록 열람실
- `/focus` = 오늘의 핵심 축 집중면

# 5. 시각 시스템 핵심 원칙

## 5.1 색의 철학

색은 화려한 전통색이 아니라, 시간의 온도 차이를 표현해야 한다.

### 기본 방향

- 배경은 먹색 중심
- 콘텐츠 표면은 회백/종이 결 계열
- 강조는 황동/금박에 가까운 저채도 호박색
- 포인트는 아주 제한적으로만 사용

### 색 사용 원칙

- 검은색은 차갑지 않고 깊어야 한다.
- 흰색은 새하얗지 않고 약간 탁해야 한다.
- 강조색은 적게 쓸수록 더 고급스럽다.
- 페이지 전체를 베이지로 덮지 않는다.

## 5.2 여백의 철학

여백은 빈 공간이 아니라, 격식과 호흡이다.

- 제목 주변 여백은 넉넉하게
- 카드 안 여백은 작지 않게
- 정보끼리 붙기보다 계층을 드러내게 분리
- 텍스트 덩어리는 숨 쉴 수 있게 유지

## 5.3 선의 철학

선은 화면을 나누는 장치이자 질서를 만드는 장치다.

- 선은 얇고 은은해야 한다.
- 너무 많은 박스보다, 선 하나로 층위를 암시할 것
- 가이드 선, 분리 선, 상단 하이라이트 선 모두 역할이 달라야 한다.

# 6. Time Descent 규칙

## 6.1 핵심 개념

페이지는 위에서 아래로 갈수록 더 과거의 질감과 기록성이 짙어진다.

## 6.2 레이어 정의

### Layer 1 — Present

- 가장 현대적
- 가장 선명한 숫자와 행동
- 대시보드 감각

### Layer 2 — Entry

- 시간축 진입
- 중심 시대 강조
- 연결 시대 프리뷰

### Layer 3 — Reconstruction

- 흐름 복원
- 연결선 / 체인 / 진단 / 메모
- 가장 서사적인 구간

### Layer 4 — Archive

- 기억 포인트
- 시험 포인트
- 보관함/인덱스 구조
- 가장 차분한 층

## 6.3 적용 질문

모든 새 페이지는 아래 질문에 답해야 한다.

1. 이 페이지의 Layer 1은 무엇인가?
2. 사용자는 어디서 시간축이나 문제 맥락에 진입하는가?
3. 복원/판단이 일어나는 핵심 구간은 어디인가?
4. 기록/정리/다음 행동으로 돌아가는 층은 어디인가?

# 8. 컴포넌트 역할 규칙

## 8.1 FrameLabel

용도:

- 공간 이름
- 섹션 정체성
- 상태 꼬리표

금지:

- 본문 요약 전체를 라벨에 몰아넣지 말 것
- 너무 긴 문장 금지

## 8.2 SectionTitle

용도:

- 섹션 입구 설명
- 상위 구조를 잡는 제목

금지:

- 설명문을 3줄 이상 길게 끌지 말 것
- 감성 카피 과다 금지

## 8.3 RecordCard

용도:

- 문제 / 오답 / 복습 항목 / 연결 단위의 핵심 기록면

구조 원칙:

- 상단 표찰
- 시대/축/분류
- 중심 제목
- 진단/흐름/메모/행동

금지:

- 일반 SaaS 카드처럼 단순 정보 박스화
- 내용이 없는데 장식만 강한 카드 금지

## 8.4 ArchivePanel

용도:

- 기록을 정리하는 최종 층
- 무엇을 남길지 보여주는 곳

구조 원칙:

- 리스트보다는 인덱스
- 정보보다 정리의 감각
- 너무 눈에 띄기보다 차분해야 함

- 향후 새 탭 설계 시 본 문서 기준으로 page role 정의 후 추가

# 16. 현재 구현 기준의 해석

현재 프론트는 아래처럼 해석한다.

- `/today`: 중심 시대와 시간축을 보여주는 진입면
- `/notes`: 오답을 기억과 진단의 기록으로 보관하는 면
- `/review`: 복습을 회복 순서와 재확인 흐름으로 다루는 면
- `/solve`: 사료와 보기 판단을 실제 학습 행동으로 연결하는 면

즉, 이 테마는 단순히 한국적인 디자인이 아니다.

이 테마는:

- 현재에서 과거로 내려가는 시간 구조를 갖고,
- 기록문화의 정갈함을 정보 구조로 번역하며,
- 학습의 회복 과정을 시각 언어로 만들고,
- 한국사 전용 학습 OS를 공통 컴포넌트로 유지하는 시스템이다.

앞으로 어떤 탭이 생겨도, 이 문서 기준으로 해석하면 같은 세계관 안에서 확장할 수 있어야 한다.
