# 한국사 판단 데스크 레퍼런스 보드

## 목적
- `/today`를 일반 생산성 대시보드가 아니라 `한국사 판단 데스크`로 설계하기 위한 공식 레퍼런스 기준선
- 한국성은 장식이 아니라 `시대 구조`, `기록물 질서`, `재질감`, `절제된 포인트`로 번역한다

## 공식 소스 우선순위
1. 시대 구조와 흐름 라벨
   - 우리역사넷 한국사연대기: https://contents.history.go.kr/front/kc/main.do
   - 우리역사넷 한국사 이미지 자료: https://contents.history.go.kr/front/ki/main.do
2. 조선적 질서감과 기록물 구조
   - 국립고궁박물관: https://www.gogung.go.kr/gogung/main/main.do
   - 왕실문화도감 일러스트: https://www.gogung.go.kr/gogung/pgm/psgudMng/view.do?psgudSn=439286&menuNo=800070
3. 시대별 재질, 색, 표면
   - e뮤지엄: https://www.emuseum.go.kr/index.html
   - 국립중앙박물관 소장품 검색: https://www.museum.go.kr/site/main/relic/search/view?relicId=8365
4. 근현대 및 일제강점기 톤
   - 대한민국역사박물관: https://www.much.go.kr/
   - 근현대사 아카이브: https://archive.much.go.kr
5. 미세한 선형 패턴 참고
   - 문화포털 전통문양 디자인가이드: https://culture.go.kr/tradition/traditionalPattern.do?type=F
   - 문화포털 전통문양 예시: https://www.culture.go.kr/tradition/patternView.do?did=24804&gubun=
6. 용어와 카피 검증
   - 한국민족문화대백과사전: https://encykorea.aks.ac.kr/

## 소스별 활용법
### 우리역사넷
- 시대 구분, 흐름명, 사건 연결 방식 참고
- UI에서 `시대 지도`, `흐름 라벨`, `끊긴 연결` 카피를 만들 때 우선 사용

### 국립고궁박물관
- 조선풍 위계, 정렬감, 기록물 문법 참고
- 아이콘은 왕실문화도감 일러스트의 `책갑`, `기록함`, `도설`, `기기` 계열 구조를 추상화

### e뮤지엄 / 국립중앙박물관
- 고려 청자, 조선 백자, 문서, 목판, 인장함처럼 시대별 재질감을 관찰
- 배경 텍스처를 쓰지 말고 `색의 깊이`, `광택 억제`, `모서리 처리`, `선 두께`로 번역

### 대한민국역사박물관
- 근현대 문서의 밀도, 흑백 대비, 긴장감 있는 레이아웃 참고
- 일제강점기와 현대 카드의 정보 배치와 대비감에 반영

### 문화포털 전통문양
- 메인 장식 금지
- divider, top strip, 반복 간격 같은 미세 리듬만 참고

## UI 적용 규칙
- 한 화면의 80% 이상은 뉴트럴 톤으로 유지
- 시대색은 `좌측 라인`, `상단 포인트 바`, `진행률 바`, `태그`, `hover/focus 상태`에만 사용
- 카드 표면은 유리판이 아니라 `기록판`처럼 보여야 함
- 한국풍은 텍스처나 기와 사진이 아니라 `위계`, `리듬`, `정렬`, `단정한 반복`으로 만든다
- `/today`의 1순위 판단과 하단 요약 판단은 절대 충돌하면 안 된다

## 카피 규칙
- 시대명만 던지지 말고 `흐름형 설명`을 함께 둔다
- 취약 주제는 `끊긴 연결`로 번역한다
- 예시
  - `경제 침탈` -> `토지 수탈 → 산업 왜곡 → 민족 경제 압박`
  - `고려 대외 관계` -> `거란 대응 → 송과의 관계 → 외교 균형`

## 구현 메모
- 현재 `today` API는 `today_review_count`, `today_new_count`, `weak_topics`만 제공
- 따라서 `누적 진척`, `연속 학습`, `시대별 실측 퍼센트`는 실제 데이터가 아니라면 전면에 두지 않는다
- 상단에는 `오늘의 판단`, `Primary Era`, `우선 작업량`만 명확하게 둔다
