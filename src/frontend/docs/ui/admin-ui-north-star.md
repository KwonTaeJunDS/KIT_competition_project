# 한국사 오답노트 RAG OS

## Admin UI North Star v1

### 1. 역할 정의

어드민은 일반 관리페이지가 아니다.

어드민은 선생님이:

- 소스를 넣고
- 개념을 추출하고
- 관계를 직접 편집하고
- 학생별로 어디서 이해가 끊겼는지 확인하고
- 어떤 설명이 전달되지 않았는지 찾는

**지식 구조 운영실**이다.

---

### 2. 1순위 가치

**직접 관계망 편집**

이 가치는 최우선으로 잠근다.

뜻:

- 자동 추출은 출발점이지만, 최종 권한은 선생님에게 있어야 한다
- 선생님은 노드와 엣지를 “검수”하는 사람이 아니라 실제로 “편집”하는 사람이어야 한다
- UI 중심은 업로드 폼이 아니라 관계망 작업면이어야 한다

---

### 3. 보조 가치

1순위는 관계망 편집이지만, 아래 가치도 동시에 유지한다.

- PDF, 정답지, 사료집에서 자동으로 개념 후보를 뽑는다
- 개념 누락, 잘못 연결된 엣지, 중복 노드를 빠르게 수정할 수 있다
- 학생별로 어떤 연결이 비었는지 바로 볼 수 있다
- 어떤 설명이 전달되지 않았는지 선생님이 판단할 수 있다

---

### 4. 어드민 UI가 줘야 하는 인상

- 정교함
- 통제감
- 밀도
- 관계성
- 추적 가능성

금지:

- 지나치게 장식적임
- 가벼운 SaaS 대시보드 톤
- 메뉴만 많은 백오피스 느낌

정답:

**개념 구조를 만지고, 학생 상태를 읽고, 다시 가르칠 지점을 결정하는 전문 도구**

---

### 5. 핵심 화면 우선순위

#### 1. 학생별 이해 누수 현황

어드민 첫 화면은 여기로 잠근다.

보여야 할 것:

- 학생별 현재 이해 상태
- 끊긴 흐름 수
- 반복 오답 개념군
- 전달 실패 가능 지점
- 다시 설명이 필요한 우선순위

즉 홈은 `지금 누가 어디서 막혔는가`를 보는 운영 대시보드다.

#### 2. 관계망 편집 화면

가장 중요하다.

보여야 할 것:

- 노드
- 엣지
- 시대
- 개념군
- 근거 소스
- 편집 상태

#### 3. 소스 업로드/자동 추출 화면

입구 역할이지만 중심은 아니다.

보여야 할 것:

- 업로드된 소스
- 추출 상태
- 후보 개념
- 검토 필요 항목

#### 4. 전달 실패 리포트

학생 데이터와 관계망을 묶어 보여주는 보조 분석층이다.

---

### 6. 시각 언어

어드민은 학생 화면보다 더 구조적이고 더 통제적이어야 한다.

원칙:

- 더 높은 정보 밀도
- 더 강한 그리드
- 더 선명한 분리선
- 더 적은 감성 문장
- 더 강한 상태/편집 피드백

시각 톤은 `교육기관용 운영 대시보드`를 기준으로 잠근다.

뜻:

- 지나치게 차갑고 기업형 분석 툴처럼 가지 않는다
- 선생님이 쓰기 쉬운 친숙한 운영 화면이어야 한다
- 하지만 단순한 학원 관리페이지보다 더 구조적이어야 한다

한국사 느낌은 학생 화면처럼 감성적으로 드러내지 않는다.
대신:

- 시대 라벨
- 개념 계층
- 관계 구조
- 사료 근거

같은 `지식 구조성`으로 드러난다.

---

### 7. 화면의 기본 메타포

어드민은 다음 메타포를 따라간다.

**운영 대시보드 + 편집기 + 지도실**

즉:

- 왼쪽은 구조
- 가운데는 작업
- 오른쪽은 근거/학생 영향

같은 식의 조합이 가장 적합하다.

---

### 8. 최종 잠금 문장

어드민 UI는

**학생별 이해 누수를 먼저 보여주고, 선생님이 한국사 개념 구조를 직접 다듬으며, 다시 가르칠 지점을 찾게 만드는 교육 운영 대시보드**여야 한다.
