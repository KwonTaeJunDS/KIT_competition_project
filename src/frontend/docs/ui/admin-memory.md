# 한국사 오답노트 RAG OS `/admin`

## 운영실 메모리 문서 v1

### 기준

- 작업 기준 프론트는 `src/frontend` 하나만 사용한다.
- 선생님 첫 화면의 1순위는 `학생별 이해 누수 현황`이다.
- 선생님의 핵심 행동은 `직접 관계망 편집`이다.
- 어드민 톤은 `교육기관용 운영 대시보드`를 기본으로 하되, 지나치게 차갑거나 분석툴처럼 보이면 안 된다.

### 제품 정의

- 이 화면은 일반 관리자 페이지가 아니라 `지식 구조 운영실`이다.
- 선생님은 여기서
  - 누가 막혔는지 보고
  - 왜 전달이 안 되었는지 파악하고
  - 관계망을 직접 고쳐
  - 학생 화면의 끊긴 흐름을 다시 잇는다.

### `/admin` 구조 잠금

1. Hero
- 제목: `학생별 이해 누수 현황`
- 설명: 학생별 누수 확인 -> 관계망 수정 -> 재설명 순서가 한 번에 읽혀야 한다.
- CTA 1: `관계망 편집 열기`
- CTA 2: `학생 누수 먼저 보기`

2. Summary Row
- 집중 관리 학생
- 전달 실패 후보
- 편집 대기 노드
- 이번 주 안정권

3. Main Left
- `지금 먼저 봐야 할 학생`
- 학생별 카드가 핵심이며, 학생 이름/반/위험도/중심 시대/끊긴 흐름/밀린 복습이 먼저 보여야 한다.
- 학생 카드의 핵심 문장은 `가장 약한 연결`과 `다음 설명 제안`이다.

4. Main Right
- `관계망 편집 진입`
- `전달 실패 신호`
- 이 두 패널은 학생 카드보다 약해야 한다.

5. Bottom
- `이번 주 먼저 설명할 개념군`
- `소스 검토 큐`

### `/admin/ontology` 구조 잠금

1. Hero
- 제목: `관계망 편집실`
- 자동 추출보다 `직접 손보는 편집`이 앞에 와야 한다.

2. 3열 작업면
- 왼쪽: `검토할 소스`
- 가운데: `지금 손볼 관계`
- 오른쪽: `학생 영향`

3. 하단
- `관계 수정 전에 볼 체크리스트`

4. 연결 규칙
- `전달 실패 신호`는 추상 리포트가 아니라 실제 `편집 작업`으로 바로 이어져야 한다.
- `소스 검토 큐`는 현재 선택한 관계 작업과 직접 연결되어야 한다.
- `학생 영향`, `전달 실패`, `소스`는 같은 작업 id를 기준으로 한 화면에서 읽혀야 한다.

### `/admin/students` 구조 잠금

1. 페이지 정체성
- `학생 현황`은 학생 목록이 아니라 `개입 우선순위 큐`다.
- 이 화면의 목적은 학생을 소개하는 것이 아니라, 오늘 누구에게 어떤 개입을 먼저 할지 5초 안에 판단하게 하는 것이다.

2. 상단
- 제목: `학생 현황`
- 보조 문구: `오늘 우선 개입이 필요한 학생과 반복 누수 축을 먼저 확인합니다.`
- 상단은 헤더 + 운영 요약 스트립 + 필터/정렬 바 3층만 유지한다.
- `TEACHER DESK`, `TODAY NOTE`, `ADMIN WORKSPACE`, 과한 소개성 문구는 운영 판단보다 앞서면 안 된다.

3. 본문 레이아웃
- 좌측 65%: 학생 우선순위 리스트
- 우측 35%: 선택 학생 상세 패널
- 전체 페이지 이동보다 같은 화면 안에서 `비교 -> 판단 -> 조치`가 먼저 일어나야 한다.

4. 좌측 리스트 규칙
- 카드 소개형보다 `고밀도 운영 행` 문법이 우선이다.
- 한 학생당 이름, 반, 위험도, 중심 시대, 끊긴 흐름, 밀린 복습, 반복 누수 축, 추천 조치, 최근 상태가 바로 보여야 한다.
- CTA는 `상세 보기`가 아니라 목적형 문구를 사용한다.
  - `누수 상세`
  - `관계축 보기`
  - `복습 큐 보기`
  - `학생 추적 보기`

5. 우측 상세 패널 규칙
- 학생 기본 정보
- 핵심 진단
- 교사용 해석
- 추천 조치 3단계
- 관련 관계망
- 최근 개입 기록
- 이 순서로 읽혀야 한다.

6. 관계망 연결 규칙
- 구조 누수 학생은 학생 리스트 단계에서 바로 `관계축 보기`로 연결돼야 한다.
- 우측 패널에는 관련 편집 task, 영향 학생 수, 편집 이동 버튼이 함께 보여야 한다.
- 학생 관리와 관계망 수정은 별도 흐름이 아니라 같은 운영 루프여야 한다.

### 시각 톤 잠금

- 배경은 밝고 따뜻한 기록판 계열
- 보더와 여백으로 신뢰감을 만든다
- 학생 페이지보다 더 정돈되고 운영실 같아야 한다
- 그러나 과한 다크 SaaS 대시보드처럼 보이면 안 된다

### 반복 금지

- 404 나는 사이드바 링크를 만들지 않는다
- 학생 화면보다 더 화려하게 만들지 않는다
- 통계 카드가 학생 카드보다 더 중요해 보이게 만들지 않는다
- 관계망 편집을 숨겨진 하위 기능처럼 다루지 않는다
- `전달 실패`를 추상적 리포트만으로 끝내지 않는다

### 다음 우선순위

1. 학생 상세 화면에서 개입 전/후 변화량 비교 추가
2. 관계망 편집실에서 노드/엣지 편집 인터랙션 프로토타입 강화
3. 학생 현황에서 같은 축 학생 묶음 보기 추가
4. 전달 실패 신호와 소스 큐를 실제 데이터 흐름과 더 촘촘히 연결
