# `/today` UI 블록 명세서 v1

컨셉: **Time Descent**  
목표: `/today`를 단순 학습 홈이 아니라, **현재의 학습자가 시간축을 따라 과거로 내려가며 끊긴 한국사 흐름을 복원하는 입구**로 고정한다.

---

## 0. 문서 목적과 범위

### 목적
- `/today` 화면의 IA, 블록 구조, 시각 위계, 상호작용을 구현 전 단계에서 잠근다.
- 디자인 논의가 반복되며 흔들리는 문제를 줄인다.
- 코딩 시작 시 바로 적용 가능한 기준선을 제공한다.

### 범위
- 대상: 학생용 `/today` 화면
- 포함: Hero, Time Rail, Learning Flow, Chain Cards, Archive Notes, Bottom Nav
- 제외: `/solve`, `/notes`, `/review`, 관리자 페이지 상세 설계

### Source of Truth
- 기준 프론트: `hanguksa_verify`
- 기준 주소: `http://localhost:4000/today`

---

## 1. 제품 역할 정의

`/today`는 다음 4가지를 동시에 수행해야 한다.

1. 오늘 가장 먼저 봐야 할 학습 대상을 명확히 제시한다.
2. 오늘 중심 시대가 전체 한국사 시간축에서 어디인지 보여준다.
3. 끊긴 연결을 `복습 -> 문제 -> 연결` 흐름으로 복원하게 만든다.
4. 다음 시대로 자연스럽게 이어지게 만든다.

즉, 이 페이지는 `대시보드 + 시간축 진입면 + 흐름 복원 입구`다.

---

## 2. 전체 레이아웃 구조

페이지는 위에서 아래로 갈수록 `현재 -> 과거`의 질감이 짙어져야 한다.

### Layer 1 — Present Dashboard
- 가장 현대적
- 숫자/KPI/CTA 선명도 최우선
- 질감 최소

### Layer 2 — Time Entry
- 시간축 진입
- 중심 시대의 위치 명확화
- 깊이감 1단계 증가

### Layer 3 — Reconstruction
- 끊긴 연결 복원 중심 구간
- 기록 조각/연결선/주석 감각 강화

### Layer 4 — Archive
- 시험 메모/대표 유물/다음 시대
- 가장 조용하고 기록물 같은 무드
- 마지막 CTA에서 행동성 회복

## 3. UI 블록 명세

## 3.1 Hero / Today Focus

### 역할
- 오늘 중심 시대 고정
- 오늘 할 일과 이유를 즉시 이해
- 복습/오답 첫 행동 제공

### 포함 정보
- 타이틀: `오늘 학습 안내`
- 서브 타이틀: `오늘 복습으로 흐름 다지기`
- 중심 시대: `일제강점기`
- 중심 축: `침탈·저항`
- 설명 1문장
- KPI 4개: 먼저 볼 복습, 이어 볼 새 문제, 끊긴 연결, 오늘 학습량
- CTA 2개: `복습 이어 보기`, `오답노트 보기`

### 시각 구조
- 좌측: 타이틀/시대/설명
- 우측: KPI 패널/CTA

### 원칙
- 전 영역 중 가장 현대적인 스타일
- 배경 텍스처 거의 없음
- 중심 시대가 가장 큰 시각 앵커
- 행동이 3초 안에 읽혀야 함

---

## 3.2 Time Rail / Historical Positioning

### 역할
- 중심 시대를 전체 한국사 흐름 속에 위치시킴
- 시대 간 연속성 인식 제공

### 포함 정보
- 섹션 제목: `시간축`
- 전체 시대: 선사, 고조선, 삼국, 남북국, 고려, 조선전기, 조선후기, 근대, 일제강점기, 현대
- 중심 시대 카드 1개
- 연결 시대 프리뷰 2개

### 구조
- 상단: 시대 레일
- 중단: 중심 시대 상세
- 하단: 연결 시대 프리뷰

### 원칙
- 단순 버튼 나열 금지
- 중심 시대만 강한 초점
- 연결 시대는 한 단계 낮은 보조 강도
- 얇고 정갈한 선형 구조

### 인터랙션
- 노드 hover: 시대 설명 미리보기
- 중심 시대 강조는 고정
- 연결 시대 클릭: 관련 흐름으로 스크롤 또는 solve 연결

---

## 3.3 Flow Steps / Study Sequence

### 역할
- 오늘 학습 순서를 명확하게 제시
- `복습 -> 새 문제 -> 다음 연결` 동선 고정

### 포함 정보
- 섹션 제목: `흐름`
- Step 01: 먼저 복습
- Step 02: 다음 문제
- Step 03: 이어서 보기

### 각 Step 구성
- 번호
- 액션명
- 소제목
- 수량
- 설명 1문장
- CTA 링크

### 원칙
- 3개 카드가 아니라 `하나의 진행 경로`처럼 보여야 함
- Step 사이 얇은 연결선 사용
- Step 01 시각 강조
- Step 03은 다음 시대 연결 다리 역할

### 모션
- 진입 시 `01 -> 02 -> 03` 순차 표시
- 과한 연출 금지, 흐름 열림만 표현

---

## 3.4 Chain Cards / Reconstruction Surface

### 역할
- 끊긴 연결 복원 핵심 구간
- 페이지에서 가장 서사적/한국사적인 영역

### 카드 포함 정보
- Chain 번호
- 시대명
- 시대 축
- 연결 주제
- 연결 사슬
- 시험에서 같이 볼 축
- 보조 설명
- notes 이동 링크

### 구조
- 상단 라벨: `Chain 01 / 일제강점기 / 침탈·저항`
- 본문: 주제 + 연결 사슬 (`토지 수탈 -> 산업 왜곡 -> 민족 경제 압박`)
- 하단: 시험 포인트 + 지금 복원해야 하는 이유

### 원칙
- `카드`보다 `기록 조각` 느낌
- 연결 사슬을 가장 큰 시각 요소로 승격
- 아이콘 버튼형 화살표 대신 정갈한 선형 구조
- 따뜻하고 깊은 톤으로 레이어 전환

### 우선순위 표현
- Chain 01 강하게
- Chain 02, 03 중간 강도

### 인터랙션
- hover: 시험 포인트 강조
- click: notes 상세 이동
- 선택 chain과 중심 시대의 시각 연결 유지

---

## 3.5 Archive Notes / Exam Memory Layer

### 역할
- 오늘 학습 기억 포인트 정리
- 시험용 기억 장치 축적
- 감정적으로 가장 조용한 레이어

### 포함 정보
- 대표 유물
- 시험 포인트
- 자주 같이 보는 흐름
- 다음 연결/다음 시대 설명
- CTA: `다음 흐름 이어 보기`

### 구조
- 좌측: 유물/시험 포인트/연계 흐름
- 우측: 다음 시대/이유/CTA

### 원칙
- 문서 인덱스/보관 라벨 감각
- 얇고 정갈한 구획선
- 과도한 카드 분절 금지
- 마지막 CTA는 행동성을 위해 현대적 버튼 톤 일부 회복

---

## 3.6 Bottom Nav / Learning Route

### 역할
- 학습 루트 지속성 제공
- 현재 위치(`오늘`) 명확히 고정

### 항목
- 오늘
- 학습
- 오답
- 복습

### 원칙
- 학생용 라우트임이 명확해야 함
- 장식보다 안정성 우선
- 하단 기록층에서 현재 행동으로 복귀시키는 균형 장치

---

## 4. 디자인 시스템 규칙

## 4.1 컬러 운영 규칙

### 레이어별 색 온도
- Layer 1: 짙은 먹색 + 높은 대비
- Layer 2: 청회색/먹색 혼합 + 중심 시대 강조
- Layer 3: 갈회색 + 탁한 황동/옅은 주홍 소량
- Layer 4: 따뜻한 회갈색 + 조용한 대비

### 강조색 규칙
- 중심 시대: 가장 선명
- 연결 시대: 중간
- 시험/기억 포인트: 저채도
- CTA: 가독성 확보를 위한 현대적 선명도 유지

### 금지
- 단청/문양 직접 사용
- 과한 베이지/장식색 확장

---

## 4.2 타이포그래피 규칙

### 원칙
- 붓글씨 폰트 금지
- 전통감은 폰트 자체가 아니라 여백/리듬/질서로 표현

### 계층
- H1: 페이지 타이틀
- H2: 시대명(핵심 앵커)
- H3: 섹션 표제
- Body: 현대 고딕, 읽기 우선
- Meta: 연도/수량/chain 번호 중심의 압축형 스타일

---

## 4.3 컴포넌트 형태 규칙

- Hero Card: 둥근 모서리 허용, 현대적 패널
- Time Rail Node: 작고 단정, 중심 노드만 강한 강조
- Flow Step: Hero보다 단단한 모서리 + 연결선
- Chain Card: 문서형 패널 + 분류표/라벨 구조
- Archive Panel: 보관함/인덱스 느낌의 큰 패널 안에 정보 조각

---

## 5. 마이크로 인터랙션 가이드

- 상단: 빠르고 또렷한 진입
- Time Rail: 부드럽고 길게 이어지는 감각
- Chain: 연결 사슬이 살짝 늦게 등장해 시선 유도
- Archive: 거의 정적, 안정감 우선
- 공통: 아래로 갈수록 모션이 더 조용해야 함

---

## 6. 카피 스타일 규칙

- 상단: 행동 유도형, 짧고 명확
- 중단: 위치/맥락 설명형
- 하단: 복원/기억형, 메모 톤

### 금지
- 긴 제품 소개 문장
- 과한 감성 카피
- 관광지/박물관식 전통 표현

---

## 7. 반응형 원칙

### Desktop
- Hero 2열
- Time Rail 가로 강조
- Chain 2~3열 가능
- Archive 좌우 분할

### Tablet
- Hero 수직 전환
- Time Rail 축소
- Chain 1~2열
- Archive 단일 컬럼

### Mobile
- Hero 요약형으로 축소
- Time Rail 세로 스택형
- Chain 1열
- Archive 접이식 블록 허용

핵심: 모바일에서도 `시간축 -> 복원 -> 기록층` 경험은 유지해야 한다.

---

## 8. 구현 우선순위 (코딩 전 고정)

1. 레이어 구조(1~4)와 섹션 순서 잠금
2. Hero/Time Rail의 정보 위계 잠금
3. Flow/Chain의 연결 표현 잠금
4. Archive/Bottom Nav의 마무리 동선 잠금
5. 반응형 전환 규칙 잠금
6. 카피 톤과 CTA 어휘 잠금

---

## 9. QA 체크리스트

- 상단 3초 내: 중심 시대/오늘 행동이 읽히는가
- Time Rail: 전체 한국사 속 현재 위치가 즉시 보이는가
- Flow: 3개 카드가 아닌 단일 경로로 느껴지는가
- Chain: 제목보다 연결 사슬이 먼저 보이는가
- Archive: 조용한 기록층으로 인식되는가
- 하단: 다음 행동(학습 라우팅)으로 자연스럽게 이어지는가
- 전체: 아래로 갈수록 화려해지지 않고 조용해지는가

---

## 10. 최종 잠금 문장

`/today`는 단순 학습 홈이 아니라,  
**현재의 학습자가 오늘의 중심 시대를 기준으로 한국사 시간축 속으로 천천히 내려가며 끊긴 흐름을 복원하는 입구**로 설계한다.
